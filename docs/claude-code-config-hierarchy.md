# Claude Code 配置/记忆文件层级

## 优先级一览

按优先级从高到低：

| 位置 | 范围 | 用途 | 优先级 |
|------|------|------|--------|
| `<项目>/.claude/CLAUDE.md` | 项目内 — 团队共享 | 随代码提交，放项目结构、构建命令、代码规范 | **最高** |
| `.claude/projects/<项目名>/memory/` | 项目级记忆 | 仅对应项目打开时加载，放上下文、用户身份、进度 | **高** |
| `.claude/memory/` | 全局记忆 — 跨项目 | MEMORY.md 索引 + .md 文件，放通用偏好/习惯/约定 | **中** |
| `~/CLAUDE.md` | 用户级 — 所有项目 | 全局指令：项目背景、QA 规范、技术备注。Claude Code 启动即加载 | **底** |

## 原则

- **越具体越优先**：所有层级同时生效，冲突时高层覆盖底层
- `.claude/memory/` 通过 `MEMORY.md` 作为索引文件，每条记忆独立一个 `.md`
- `.claude/projects/<项目名>/` 按项目路径自动映射（如 `C--Users-evil-Desktop-project-personal-planner`）
- `~/CLAUDE.md` 属用户级兜底，优先级最低，但作用域最广

## 四种位置区别

| 维度 | `<项目>/.claude/CLAUDE.md` | `.claude/projects/<项目>/memory/` | `.claude/memory/` | `~/CLAUDE.md` |
|------|---------------------------|----------------------------------|-------------------|---------------|
| **形式** | 项目根目录下的单文件 | 目录 + MEMORY.md 索引 + 多条 `.md` | 目录 + MEMORY.md 索引 + 多条 `.md` | 用户目录下的单文件 |
| **作用范围** | 仅此项目 | 仅此项目 | 所有项目 | 所有项目 |
| **管理者** | 用户手动编辑，随代码提交 | Claude 自动写入/更新/删除 | Claude 自动写入/更新/删除 | 用户手动编辑 |
| **适合放** | 项目结构、构建命令、代码规范（团队共享） | 项目上下文、用户身份、开发进度（项目私有） | 通用偏好、习惯、约定（跨项目复用） | 项目背景、技术备注、静态参考（全局兜底） |
| **规模** | 一个文件，随代码版本管理 | 索引行简短，可无限扩展 | 索引行简短，可无限扩展 | 一个文件，太长会挤占上下文 |
| **稳定性** | 偶尔更新，通常不变 | 频繁增删，随对话积累 | 偶尔增删，积累规律性内容 | 改得少，长期稳定 |
| **版本管理** | Git 跟踪，团队共享 | 不入 Git，Claude 内部使用 | 不入 Git，Claude 内部使用 | 不入 Git，个人使用 |
| **示例** | 项目 README 技术栈说明、lint 规则 | "正在实现 XX 功能"、"用户偏好 YY 风格" | GitHub 推送流程、QA 档案规范 | vlog-metaverse/qabot 项目概览 |
